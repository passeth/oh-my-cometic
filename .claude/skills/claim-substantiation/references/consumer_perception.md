# Consumer Perception Study Guide

화장품 클레임 입증을 위한 소비자 인식 조사 방법론입니다.

## 1. Home Use Test (HUT) 설계

### HUT 개요

Home Use Test는 실제 가정환경에서 제품을 사용한 후 소비자 평가를 수집하는 방법입니다.

**목적**:
- 실제 사용 조건에서의 제품 평가
- 소비자 언어 기반 클레임 개발
- 사용감 및 만족도 측정
- 재구매 의향 파악

### HUT 유형

#### Monadic Test (단일 제품 평가)
```
구조: 1개 제품만 평가
장점: 단순, 비교 영향 없음
단점: 기준점 부재
적용: 신제품 절대 평가
```

#### Sequential Monadic Test (순차 단독 평가)
```
구조: 2개 제품을 순차적으로 각각 평가
순서: A → 세척 기간 → B (또는 B → A)
장점: 비교 가능, 순서 효과 통제
적용: 제품 비교, 개선 제품 평가
```

#### Paired Comparison Test (직접 비교 평가)
```
구조: 2개 제품 동시 제공, 선호도 비교
적용: 직접 비교 클레임
"10명 중 7명이 선호"
```

### HUT 프로토콜

#### 1단계: 설계

```
목적 정의:
- 평가할 클레임 명확화
- 측정 항목 결정
- 통계적 목표 설정

제품 준비:
- 블라인드 패키징 (필요시)
- 충분한 양 (사용 기간 + 여유분)
- 사용 설명서 동봉

일정 수립:
- 사용 기간: 2-8주 (클레임에 따라)
- 중간 체크: 2주 간격
- 최종 평가: 사용 종료 후
```

#### 2단계: 피험자 모집

**모집 기준**:
```
일반 기준:
- 타겟 연령층 (예: 30-50대 여성)
- 제품 카테고리 사용자
- 비경쟁사 직원

특정 기준 (클레임별):
- 건성 피부: "피부가 건조하다" 자가 보고
- 민감 피부: "피부가 민감하다" 자가 보고
- 안티에이징: "주름이 신경 쓰인다" 자가 보고
```

**피험자 수**:
```
통계적 유의성 확보:
- 최소: 30명
- 권장: 50-100명
- 하위 그룹 분석 시: 그룹당 30명 이상

탈락률 고려:
- 예상 탈락률: 10-20%
- 실제 모집: 목표 × 1.2
```

#### 3단계: 제품 배포

```
배포 방법:
- 대면 전달 (권장)
- 택배 발송

배포 시 포함 내용:
- 시험 제품 (라벨 코드화)
- 사용 설명서
- 사용 일지 (선택)
- 연락처 안내
- 중간/최종 설문 일정 안내
```

#### 4단계: 사용 및 모니터링

```
모니터링 방법:
- 중간 전화/문자 연락
- 온라인 중간 설문
- 사용 일지 수집 (선택)

확인 사항:
- 제품 사용 여부
- 이상 반응 발생 여부
- 질문 응대
```

#### 5단계: 최종 평가 수집

```
수집 방법:
- 온라인 설문 (가장 일반적)
- 전화 인터뷰
- 대면 인터뷰

수집 시점:
- 사용 종료 직후 (1-3일 이내)
```

## 2. 설문 문항 구성

### 문항 유형

#### 리커트 척도 (Likert Scale)

**5점 척도** (가장 일반적)
```
1 = 전혀 그렇지 않다
2 = 그렇지 않다
3 = 보통이다
4 = 그렇다
5 = 매우 그렇다
```

**7점 척도** (더 세밀한 구분)
```
1 = 전혀 동의하지 않음
2 = 동의하지 않음
3 = 약간 동의하지 않음
4 = 중립
5 = 약간 동의함
6 = 동의함
7 = 매우 동의함
```

#### Top Box / Top 2 Box

```
Top Box: 최상위 응답 비율
예: "매우 그렇다" 응답 비율

Top 2 Box: 상위 2개 응답 비율
예: "그렇다" + "매우 그렇다" 응답 비율

클레임 표현: "90%가 동의" = Top 2 Box 90%
```

#### VAS (Visual Analog Scale)

```
0 |-------|-------|-------|-------| 10
전혀 그렇지 않다          매우 그렇다

장점: 연속형 데이터, 세밀한 측정
단점: 해석이 어려움
```

### 영역별 설문 문항 예시

#### 효능 평가

**보습**
```
Q1. 이 제품 사용 후 피부가 촉촉해졌다.
    [ ] 전혀 그렇지 않다 ~ [ ] 매우 그렇다

Q2. 이 제품은 하루 종일 보습력을 유지해 준다.
    [ ] 전혀 그렇지 않다 ~ [ ] 매우 그렇다

Q3. 이 제품 사용 후 피부 당김이 개선되었다.
    [ ] 전혀 그렇지 않다 ~ [ ] 매우 그렇다
```

**주름개선**
```
Q1. 이 제품 사용 후 피부가 탄력 있어 보인다.
Q2. 이 제품 사용 후 잔주름이 개선되었다.
Q3. 이 제품은 피부를 팽팽하게 해준다.
```

**미백/톤업**
```
Q1. 이 제품 사용 후 피부톤이 밝아졌다.
Q2. 이 제품은 칙칙한 피부를 환하게 해준다.
Q3. 이 제품 사용 후 피부가 맑아 보인다.
```

**진정**
```
Q1. 이 제품은 예민해진 피부를 진정시켜 준다.
Q2. 이 제품 사용 후 피부 자극이 줄어들었다.
Q3. 이 제품은 민감한 피부에 자극이 없다.
```

#### 사용감 평가

```
Q1. 이 제품의 발림성이 좋다.
Q2. 이 제품은 빠르게 흡수된다.
Q3. 이 제품 사용 후 끈적임이 없다.
Q4. 이 제품의 향이 좋다.
Q5. 이 제품의 제형이 마음에 든다.
```

#### 전반적 만족도

```
Q1. 이 제품에 전반적으로 만족한다.
    [ ] 전혀 그렇지 않다 ~ [ ] 매우 그렇다

Q2. 이 제품을 다른 사람에게 추천하겠다.
    [ ] 전혀 그렇지 않다 ~ [ ] 매우 그렇다

Q3. 이 제품을 다시 구매할 의향이 있다.
    [ ] 전혀 그렇지 않다 ~ [ ] 매우 그렇다

Q4. 이 제품의 가격 대비 가치가 좋다.
    [ ] 전혀 그렇지 않다 ~ [ ] 매우 그렇다
```

#### 비교 평가 (해당 시)

```
Q1. 기존에 사용하던 제품과 비교했을 때, 이 제품이 더 좋다.
Q2. 이 제품은 다른 [카테고리] 제품보다 [효능]에 효과적이다.
```

### 설문 구성 원칙

**문항 순서**
```
1. 스크리닝 질문 (적격성 확인)
2. 사용 확인 질문
3. 효능 평가 (핵심 클레임 관련)
4. 사용감 평가
5. 전반적 만족도
6. 비교 평가 (해당 시)
7. 주관식 의견 (선택)
8. 인구통계 정보
```

**문항 작성 원칙**
```
- 한 문항에 하나의 개념만 질문
- 유도 질문 피하기
- 이중 부정 피하기
- 전문 용어 대신 일상 언어 사용
- 적절한 문항 수 (15-25문항 권장)
```

## 3. 통계적 유의성 확보

### 필요 표본 크기 계산

**비율 추정 (한 그룹)**
```
n = Z² × p × (1-p) / e²

n: 필요 표본 크기
Z: 신뢰수준 Z값 (95% → 1.96)
p: 예상 비율 (모를 경우 0.5)
e: 허용 오차

예시: 95% 신뢰수준, ±10% 오차
n = 1.96² × 0.5 × 0.5 / 0.1² = 96명
```

**클레임별 권장 표본 크기**

| 클레임 유형 | 최소 | 권장 | 정밀 |
|------------|------|------|------|
| 단순 만족도 | 30 | 50 | 100 |
| 비율 클레임 | 50 | 100 | 200 |
| 비교 클레임 | 60 | 100 | 150 |
| 하위 그룹 분석 | 그룹당 30 | 그룹당 50 | 그룹당 100 |

### 통계 분석 방법

#### 단일 비율 검정

**클레임**: "80% 이상이 효과에 동의"

```python
# One-sample proportion test
from statsmodels.stats.proportion import proportions_ztest

# 100명 중 85명이 동의
count = 85
nobs = 100
value = 0.80  # 검정할 비율

stat, p_value = proportions_ztest(count, nobs, value, alternative='larger')

# p < 0.05이면 "80% 이상" 클레임 가능
```

#### 두 비율 비교

**클레임**: "기존 제품보다 선호도 높음"

```python
# Two-sample proportion test
count = np.array([75, 60])  # 각 제품 선호 인원
nobs = np.array([100, 100])  # 각 제품 평가 인원

stat, p_value = proportions_ztest(count, nobs, alternative='two-sided')
```

#### 평균 비교

**사용 전후 비교 (Paired t-test)**
```python
from scipy import stats

before = [평균 점수 리스트]
after = [평균 점수 리스트]

t_stat, p_value = stats.ttest_rel(before, after)
```

**두 제품 비교 (Independent t-test)**
```python
product_A = [점수 리스트]
product_B = [점수 리스트]

t_stat, p_value = stats.ttest_ind(product_A, product_B)
```

### 신뢰구간 계산

**비율의 신뢰구간**
```
CI = p ± Z × √(p(1-p)/n)

예시: 100명 중 85명 동의 (p=0.85)
95% CI = 0.85 ± 1.96 × √(0.85×0.15/100)
       = 0.85 ± 0.07
       = [0.78, 0.92]

클레임: "78-92%가 동의" 또는 "약 85%가 동의"
```

## 4. 클레임 표현 전환

### 설문 결과 → 클레임 전환 가이드

#### 비율 클레임

| 설문 결과 | 클레임 표현 |
|----------|-----------|
| Top 2 Box 90% | "사용자 10명 중 9명이 동의" |
| Top 2 Box 85% | "85%가 효과를 체감" |
| Top Box 70% | "10명 중 7명이 매우 만족" |

**주의사항**:
- 반올림 기준 명시 (예: 87.5% → 88% 또는 약 90%)
- 표본 크기 명시 (n=xx)
- 조사 기간 명시 (xx주 사용 후)

#### 비교 클레임

| 설문 결과 | 클레임 표현 |
|----------|-----------|
| 선호도 70% vs 30% | "기존 제품 대비 선호도 70%" |
| 평균 점수 4.2 vs 3.5 | "만족도 20% 향상" |

**주의사항**:
- 비교 대상 명시 필요
- 통계적 유의성 확보 필수

### 클레임 문구 작성 예시

**보습 클레임**
```
설문 결과: "피부가 촉촉해졌다" Top 2 Box 92%
클레임: "사용자 92%가 촉촉해진 피부를 체감했습니다*"
주석: *자사 소비자 조사 결과, n=100, 4주 사용 후
```

**주름개선 클레임**
```
설문 결과: "피부 탄력이 개선되었다" Top 2 Box 88%
클레임: "10명 중 9명이 피부 탄력 개선을 느꼈습니다*"
주석: *자사 소비자 조사 결과, n=50, 8주 사용 후
```

**재구매 의향 클레임**
```
설문 결과: "다시 구매하겠다" Top 2 Box 85%
클레임: "85% 재구매 의향*"
주석: *자사 소비자 조사, n=100
```

### 클레임 표현 주의사항

**허용 표현**
```
- "~%가 동의했습니다"
- "~%가 효과를 체감했습니다"
- "~%가 개선을 느꼈습니다"
- "사용자 조사 결과"
```

**주의 표현**
```
- "임상 시험 결과" → 의료 시험 아닐 경우 혼동
- "과학적으로 입증" → 실제 과학 시험 필요
- "100%가 만족" → 현실적이지 않음
- "모든 사용자가" → 과장
```

**필수 주석 정보**
```
- 조사 유형: "자사 소비자 조사"
- 표본 크기: "n=100"
- 사용 기간: "4주 사용 후"
- 조사 시점: "2024년 x월" (선택)
```

## 5. HUT 수행 비용 및 일정

### 예상 비용 (2024년 기준)

| 항목 | 비용 범위 | 비고 |
|------|----------|------|
| 설문 설계 | 50-100만원 | 전문 기관 의뢰 시 |
| 피험자 모집 | 인당 1-3만원 | 모집 난이도 비례 |
| 제품 제공 | 제품 원가 | 충분한 양 |
| 인센티브 | 인당 3-10만원 | 사용 기간 비례 |
| 설문 수집/분석 | 50-150만원 | 문항 수, 분석 복잡도 |
| 보고서 작성 | 50-100만원 | 포함/별도 |

**예시: 100명, 4주 HUT**
```
- 피험자 모집: 100명 × 2만원 = 200만원
- 인센티브: 100명 × 5만원 = 500만원
- 설문 설계/분석: 100만원
- 보고서: 50만원
- 제품: 100개 × 원가
----------------------------------------
총 예상 비용: 약 850만원 + 제품비
```

### 일정 예시

```
Week 1: 설문 설계 및 검토
Week 2: 피험자 모집
Week 3: 제품 배포, 사용 시작
Week 4-6: 사용 기간 (4주)
Week 7: 최종 설문 수집
Week 8: 데이터 정리 및 분석
Week 9: 보고서 작성
----------------------------------------
총 소요 기간: 약 9주 (2개월)
```
